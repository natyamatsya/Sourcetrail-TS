# Roadmap: Proc-Macro Expansion for the Rust Indexer

## Goal

Enable the Rust indexer to expand procedural macros during HIR analysis so that
symbols generated by `#[derive(...)]`, attribute macros, and bang macros are
visible in the Sourcetrail graph — just as they are in rust-analyzer itself.

Without proc-macro expansion, items that only exist after expansion (e.g. the
`Debug::fmt` impl generated by `#[derive(Debug)]`, or the entire body of a
`tokio::main`-annotated function) are invisible to the indexer.

---

## Background: How ra_ap_hir Handles Proc-Macros

`load_workspace_at()` accepts a `LoadCargoConfig` with a `with_proc_macro_server`
field of type `ProcMacroServerChoice`:

```rust
pub enum ProcMacroServerChoice {
    Sysroot,           // find rust-analyzer-proc-macro-srv in the active toolchain
    Explicit(AbsPathBuf), // path to a specific proc-macro-srv binary
    None,              // disable proc-macro expansion (current behaviour)
}
```

When `Sysroot` or `Explicit` is chosen, `load_workspace_at()`:

1. Spawns `rust-analyzer-proc-macro-srv` as a subprocess.
2. Loads each proc-macro dylib (`.so` / `.dylib` / `.dll`) listed in
   `cargo metadata` build script output.
3. Registers expanders in the `RootDatabase` so that `ra_ap_hir` expands
   attribute and derive macros transparently during HIR construction.

The proc-macro server binary ships with rust-analyzer and is also available
standalone via `rustup component add rust-analyzer`.

## Current State

```rust
// src/rust_indexer/indexer/src/parser/mod.rs (ra_ap_* 0.0.341)
let load_config = LoadCargoConfig {
    load_out_dirs_from_check: false,
    with_proc_macro_server: ProcMacroServerChoice::None,  // ← expansion disabled
    prefill_caches: false,
    proc_macro_processes: 1,
    num_worker_threads: 1,
};
```

Proc-macro dylibs are not loaded, so:

- `#[derive(Debug, Clone, …)]` impls are not indexed.
- `#[tokio::main]`, `#[async_trait]`, etc. are treated as opaque.
- Bang macros (`vec![]`, `format!()`, …) from external crates are not expanded.

## Phase PM-1 — Locate the Proc-Macro Server

**Goal:** reliably find `rust-analyzer-proc-macro-srv` at runtime without
requiring the user to install rust-analyzer separately.

### PM-1 Tasks

- [ ] Try `ProcMacroServerChoice::Sysroot` first — `ProjectWorkspace::find_sysroot_proc_macro_srv()`
      searches the active toolchain's sysroot for `rust-analyzer-proc-macro-srv`.
      This works if the user has `rustup component add rust-analyzer` installed.
- [ ] Fall back to searching `PATH` for `rust-analyzer-proc-macro-srv` (the
      standalone binary from a system rust-analyzer install).
- [ ] Add a CLI flag `--proc-macro-srv <path>` to `sourcetrail_rust_indexer`
      so the C++ app can pass an explicit path (set in project settings UI).
- [ ] If no server is found, log a warning and continue with
      `ProcMacroServerChoice::None` (graceful degradation — current behaviour).

### Discovery Logic (sketch)

```rust
fn find_proc_macro_srv() -> ProcMacroServerChoice {
    // 1. Explicit CLI override
    if let Some(path) = cli_args.proc_macro_srv {
        return ProcMacroServerChoice::Explicit(AbsPathBuf::assert(path));
    }
    // 2. Sysroot (rustup component add rust-analyzer)
    if let Some(Ok(path)) = workspace.find_sysroot_proc_macro_srv() {
        return ProcMacroServerChoice::Explicit(path);
    }
    // 3. PATH
    if let Ok(path) = which::which("rust-analyzer-proc-macro-srv") {
        return ProcMacroServerChoice::Explicit(AbsPathBuf::assert(path));
    }
    ProcMacroServerChoice::None
}
```

**New dependency:** `which = "6"` (locate binaries on PATH).

## Phase PM-2 — Build Script Output (`OUT_DIR`)

**Goal:** run `cargo check` to collect build script output so that proc-macro
dylib paths are known before loading the workspace.

### PM-2 Background

Proc-macro crates are compiled to dylibs by `cargo build`. Their paths are
recorded in `cargo metadata` / build script output under
`$OUT_DIR/../../../deps/libmy_macro-<hash>.dylib`. `ra_ap_load_cargo` can
discover these automatically when `load_out_dirs_from_check: true`.

### PM-2 Tasks

- [ ] Set `load_out_dirs_from_check: true` in `LoadCargoConfig`.
      This runs `cargo check --message-format=json` internally and collects
      `OUT_DIR` values and artifact paths for all build scripts.
- [ ] Handle the case where `cargo check` fails (e.g. missing dependencies):
      log the error from `workspace.run_build_scripts()` and continue without
      proc-macro expansion rather than aborting indexing.
- [ ] Cache the build script output between indexer runs. `ra_ap_load_cargo`
      does not persist this itself — consider writing the `cargo check` JSON
      output to a file in `userDataPath` keyed by crate root + `Cargo.lock`
      hash, and re-using it if the lock file has not changed.

### Expected `LoadCargoConfig`

```rust
LoadCargoConfig {
    load_out_dirs_from_check: true,   // ← enables build script execution
    with_proc_macro_server: find_proc_macro_srv(),
    prefill_caches: false,
    proc_macro_processes: num_cpus::get().min(4),
}
```

**New dependency:** `num_cpus = "1"` (set sensible parallelism).

## Phase PM-3 — Indexing Expanded Items

**Goal:** walk HIR items that only exist after proc-macro expansion and emit
them into `IntermediateStorage`.

### PM-3 What Expansion Adds

| Macro kind | Example | Items generated |
| --- | --- | --- |
| `#[derive(Debug)]` | `struct Foo;` | `impl Debug for Foo { fn fmt(…) }` |
| `#[derive(Clone)]` | `struct Foo;` | `impl Clone for Foo { fn clone(…) }` |
| `#[async_trait]` | `trait MyTrait` | rewrites all `async fn` bodies |
| `#[tokio::main]` | `async fn main()` | wraps body in `Runtime::block_on` |
| `vec![]`, `format!()` | call sites | expands to inline expressions |

### PM-3 Tasks

- [ ] In `collect_crate()`, also walk `Module::impl_defs(db)` to pick up
      `impl` blocks (including derive-generated ones). For each `Impl`:
  - Emit an `EDGE_INHERITANCE` if it is a trait impl (`impl Trait for Type`).
  - Walk `Impl::items(db)` for associated functions/types/consts and emit them
    as `NODE_METHOD` / `NODE_TYPEDEF` / `NODE_GLOBAL_VARIABLE` nodes.
- [ ] Distinguish macro-generated impls from hand-written ones:
      check `Impl::source(db)` — if `in_file.file_id` is a macro file
      (`HirFileId::macro_file().is_some()`), tag the node with
      `DefinitionKind::IMPLICIT` (= 1) instead of `EXPLICIT` (= 2).
- [ ] Emit `EDGE_MACRO_USAGE` occurrences linking the derive attribute site
      (e.g. `#[derive(Debug)]` token range) to the `Debug` trait node.
- [ ] Add `EDGE_CALL` edges from expanded macro bodies to the functions they
      call (requires walking `Body` via `ra_ap_hir::Function::body(db)` —
      deferred to a later phase if too expensive).

### PM-3 `collect_impl` Sketch

```rust
fn collect_impl(&mut self, imp: ra_ap_hir::Impl) {
    // Trait impl → EDGE_INHERITANCE
    if let Some(trait_) = imp.trait_(self.db) {
        let type_name = imp.self_ty(self.db).display(self.db).to_string();
        let trait_name = trait_.name(self.db).as_str().to_string();
        self.emit_inheritance_edge(&type_name, &trait_name, imp.source(self.db));
    }
    // Associated items
    for item in imp.items(self.db) {
        match item {
            AssocItem::Function(f) => { /* emit NODE_METHOD */ }
            AssocItem::Const(c)    => { /* emit NODE_GLOBAL_VARIABLE */ }
            AssocItem::TypeAlias(t) => { /* emit NODE_TYPEDEF */ }
        }
    }
}
```

## Risk & Mitigation

| Risk | Mitigation |
| --- | --- |
| `cargo check` is slow (seconds per crate) | Cache build script output keyed by `Cargo.lock` hash; skip if unchanged |
| Proc-macro server not installed | Graceful degradation to `ProcMacroServerChoice::None` with a user-visible warning |
| Proc-macro dylib ABI mismatch (wrong rustc version) | `ProcMacroClient::spawn()` validates the dylib ABI; log and skip mismatched dylibs |
| Recursive / infinite macro expansion | `ra_ap_hir` has a built-in expansion depth limit; no extra work needed |
| Expanded items bloat the graph | Filter by `is_local` (same check as today) — expanded items in external crates are skipped |
